### 4.5 功能一：关键词关联与词条注入 ✅

#### 核心思路

在每次调用大模型前，`keyword_service` 扫描当前输入文本（大纲或前文末尾），匹配世界观词条库中的触发规则，将命中词条按权重排序后拼入 System Prompt。

#### 匹配规则

每条词条支持两种触发方式：

| 触发类型 | 示例 | 说明 |
|----------|------|------|
| 关键词匹配 | `["叶辰", "主角"]` | 命中任意一个词即触发 |
| 正则匹配 | `r"第[一二三四五六七八九十百]+重"` | 用于修炼境界等复杂模式 |

#### 注入策略

```
输入文本
  → keyword_service.match(text)
  → 返回命中词条列表（按 weight 降序）
  → 按 token 预算截断（防止注入过多）
  → 拼入 System Prompt 的"世界观设定"区块
```

**注入格式示例（System Prompt 片段）：**
```
[世界观设定 - 自动注入]
【叶辰】主角，天灵根，当前境界：筑基期三层...
【星辰大陆】故事发生地，魔法与武技并存...
```

#### 词条权重说明

- `weight` 字段范围：1 ~ 10，数值越高越优先注入
- 当命中词条总 token 超出预算时，低权重词条优先裁剪
- 权重相同时按词条最近更新时间排序

### 4.6 功能二：章节幕结构 ✅

#### 层级关系

```
小说 (Novel)
 └── 章节 (Chapter)
      ├── chapter_outline   # 单章概要（用户编写或 AI 草拟）
      └── 幕 (Act) × N
           ├── act_outline  # 幕概要（用户编写或 AI 草拟）
           └── content      # 幕正文（AI 按幕概要扩写）
```

#### 创作流程

```
① 用户为章节编写"章节概要"
      ↓
② 用户（或 AI 辅助）拆分出若干"幕"，并为每幕写"幕概要"
      ↓
③ 对每一幕，点击"扩写" → 调用 /api/acts/{id}/expand
      ↓
④ 后端组装 Prompt：
     System: 世界观设定（关键词注入）+ 写作风格要求
     User:   章节概要 + 前序各幕概要 + 本幕概要 + 前幕末尾N字
      ↓
⑤ 流式返回扩写正文，填充到幕的 content 字段
```

#### Prompt 组装示意

```
[System]
你是一名小说写作助手，风格：{style}
[世界观设定]
{keyword_injected_entries}

[User]
本章概要：{chapter_outline}
前情幕概要：
  第1幕：{act1_outline}
  第2幕：{act2_outline}
  ...
当前幕（第N幕）概要：{current_act_outline}
前幕结尾（最后300字）：{prev_act_tail}

请根据以上信息扩写第N幕的完整正文。
```

### 4.7 功能三：多模型支持 ✅

#### 模型配置

系统维护一张 `model_configs` 表，每条记录代表一个可用模型实例，支持动态增删。

| 配置项 | 说明 |
|--------|------|
| `provider` | 提供商（openai / anthropic / deepseek / 自定义） |
| `model_name` | 模型标识（如 `gpt-4o`、`claude-sonnet-4-5`） |
| `api_key` | 加密存储 |
| `base_url` | 支持自定义代理或本地部署地址 |
| `max_tokens` | 单次最大生成 token 数 |
| `is_active` | 是否启用 |
| `label` | 用户自定义显示名（如"写作专用 GPT-4"） |

#### 调用路由

`llm_service` 根据请求中传入的 `model_config_id` 动态选择 SDK 和参数：

```python
# 伪代码
def get_llm_client(model_config_id):
    config = db.get(ModelConfig, model_config_id)
    if config.provider == "openai":
        return OpenAI(api_key=config.api_key, base_url=config.base_url)
    elif config.provider == "anthropic":
        return Anthropic(api_key=config.api_key)
    elif config.provider == "deepseek":
        return OpenAI(api_key=config.api_key, base_url="https://api.deepseek.com")
    # ...
```

#### 前端交互

- AI 操作面板（AIPanel）顶部提供模型下拉选择器
- 用户选择后，该选择持久化到 localStorage，下次打开默认复用
- 不同操作（扩写 / 改写）可独立记忆上次选用的模型

## 8. 部署方案

### 8.1 部署方式 ✅

**本地单机部署，直接裸跑，无需 Docker。**

```bash
# 后端启动
cd backend
pip install -r requirements.txt
alembic upgrade head        # 初始化 SQLite 数据库
uvicorn main:app --reload --port 8000

# 前端启动
cd frontend
npm install
npm run dev                 # 访问 http://localhost:5173
```

数据库文件默认路径：`backend/novel.db`（单文件，直接复制即可备份）

### 8.2 服务一览

| 服务 | 启动方式 | 地址 |
|------|----------|------|
| 前端 | `npm run dev` | http://localhost:5173 |
| 后端 | `uvicorn main:app` | http://localhost:8000 |
| 数据库 | SQLite 文件，自动创建 | `backend/novel.db` |
| ~~PostgreSQL~~ | 已移除 | — |
| ~~Redis~~ | 已移除 | — |

### 8.3 Docker 方案（可选）

如果后续希望一键启动或打包分发，可以用简化版 docker-compose，仅保留两个容器：

```yaml
# docker-compose.yml（简化版）
services:
  backend:
    build: ./backend
    ports:
      - "8000:8000"
    volumes:
      - ./data:/app/data    # SQLite 文件持久化

  frontend:
    build: ./frontend
    ports:
      - "5173:5173"
```

### 8.4 数据备份

直接复制 `novel.db` 文件即可完成全量备份，无需 pg_dump 等工具。
